Introducción

La asignatura de Bases de Datos se centra en los principios y técnicas para almacenar, gestionar y recuperar información de manera eficiente y estructurada. Es un tema fundamental en el campo de la informática, ya que casi todas las aplicaciones modernas dependen de bases de datos para manejar grandes volúmenes de datos.

Modelo Relacional de Bases de Datos

  • Teorías de bases de datos relacionales: Entidades, atributos, relaciones, dominios y tuplas.
  • Relaciones entre tablas: Conceptos de claves primarias, claves foráneas, relaciones uno a uno, uno a muchos y muchos a muchos.
  • Normalización: Conceptos de normalización (1NF, 2NF, 3NF, BCNF), eliminación de redundancias y dependencias funcionales.
  • Desnormalización: Cuándo y por qué desnormalizar una base de datos para mejorar el rendimiento.

Lenguaje de Consulta Estructurado (SQL)

  • Introducción a SQL: Lenguaje estándar para la gestión y consulta de bases de datos.
  • Consultas básicas:
    • SELECT: Cómo recuperar datos de una o varias tablas.
    • WHERE: Filtrar registros según condiciones.
    • JOIN: Combinar datos de varias tablas mediante claves foráneas.
    • ORDER BY: Ordenar los resultados de las consultas.
    • GROUP BY: Agrupar los datos según una o más columnas.
  • Modificación de datos:
    • INSERT: Insertar nuevos registros en una tabla.
    • UPDATE: Modificar registros existentes.
    • DELETE: Eliminar registros de una tabla.
  • Funciones agregadas: SUM, COUNT, AVG, MAX, MIN.
  • Subconsultas y vistas: Cómo usar subconsultas dentro de otras consultas y la creación de vistas para abstracción.
  • Índices y optimización de consultas: Cómo crear índices y su impacto en el rendimiento de las consultas.

Diseño de Bases de Datos

  • Modelo Entidad-Relación (ER): Cómo diseñar bases de datos utilizando diagramas ER, representando entidades, atributos y relaciones.
  • Conversión de un diagrama ER a un modelo relacional: Traducir diagramas de entidad-relación a tablas y relaciones.
  • Restricciones de integridad: Integridad de entidad, referencial y de dominio.
  • Caso práctico de diseño: Diseño de una base de datos para un sistema real (p.ej. biblioteca, ventas, etc.).

Transacciones y Control de Concurrencia

  • Transacciones en bases de datos: Propiedades ACID (Atomicidad, Consistencia, Aislamiento, Durabilidad).
  • Control de concurrencia: Bloqueos, control optimista, control pesimista.
  • Rollback y Commit: Cómo deshacer o confirmar cambios en la base de datos durante una transacción.

Seguridad en Bases de Datos

  • Autenticación y autorización: Control de acceso, roles y permisos.
  • Encriptación de datos: Proteger datos sensibles mediante cifrado.
  • Respaldo y recuperación: Copia de seguridad (backups) y restauración en caso de fallo.

Bases de Datos No Relacionales (NoSQL)

  • Introducción a NoSQL: Comparación con las relacionales, cuándo usar cada una.
  • Tipos de bases de datos NoSQL:
    • Documentales: MongoDB
    • Clave-Valor: Redis
    • Columnar: Cassandra
    • Grafos: Neo4j
  • Consultas en NoSQL: Cómo consultar bases NoSQL y sus limitaciones.

Optimización y Rendimiento

  • Optimización de consultas SQL: Uso de índices, consultas eficientes.
  • Manejo de grandes volúmenes de datos: Sharding, particionamiento.
  • Caché de bases de datos: Uso de cachés para mejorar el rendimiento.

Bases de Datos Distribuidas

  • Arquitectura distribuida: Cómo las BD pueden estar en múltiples servidores.
  • Replicación y particionamiento: Técnicas para distribuir datos y asegurar consistencia.
  • Teorema CAP: Consistencia, disponibilidad y partición en sistemas distribuidos.

Prácticas y Laboratorio

  • Diseño de bases de datos: Creación a partir de casos prácticos.
  • Desarrollo de consultas SQL: Resolución de consultas complejas.
  • Transacciones y concurrencia: Experimentos prácticos para manejar transacciones.
  • Trabajo con SGBD: MySQL, PostgreSQL, MongoDB, Redis.
  • Proyecto final: App completa que interactúe con BD relacionales y/o NoSQL.

Herramientas y Tecnología Utilizadas

  • Lenguajes: SQL, MongoDB query language, etc.
  • SGBD Relacionales: MySQL, PostgreSQL, Oracle, SQL Server.
  • SGBD No Relacionales: MongoDB, Redis, Cassandra, Neo4j.
  • Herramientas de administración: phpMyAdmin, pgAdmin, MongoDB Compass, Robo 3T.
