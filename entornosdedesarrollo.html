Introducción

La asignatura de Entornos de Desarrollo se centra en proporcionar las habilidades y conocimientos necesarios para trabajar con herramientas y entornos de desarrollo de software. A través de esta materia, los estudiantes aprenden a configurar, gestionar y utilizar las herramientas necesarias para desarrollar, probar y desplegar aplicaciones.

Qué vamos a aprender

  • Concepto de entorno de desarrollo: Definición, importancia y tipos de entornos en el ciclo de vida del software (entorno de desarrollo, pruebas, producción).
  • Tipos de entornos de desarrollo:
    • IDE (Entorno de Desarrollo Integrado) vs. Editores de texto avanzados.
    • Herramientas y plataformas para diferentes lenguajes (Java, Python, C#, etc.).

Instalación y configuración de entornos de desarrollo

  • Configuración de un IDE: Configuración de herramientas como Eclipse, NetBeans, IntelliJ IDEA y Visual Studio Code.
  • Configuración de variables de entorno: PATH y otras variables necesarias para la correcta ejecución de compiladores e intérpretes.
  • Gestión de plugins y extensiones:Cómo instalar y configurar plugins que faciliten el desarrollo (por ejemplo, plugins de control de versiones, depuración, y soporte para lenguajes adicionales).
  • Creación y configuración de proyectos: Configuración inicial y estructura de proyectos en los distintos IDEs.

Control de versiones

  • Introducción a los sistemas de control de versiones: Qué es el control de versiones y por qué es fundamental en el desarrollo de software.
  • Git y GitHub/GitLab:
    • Conceptos básicos de Git: commit, push, pull, branch, merge, etc.
    • Creación de repositorios locales y remotos.
    • Gestión de ramas para trabajar en paralelo y fusionar cambios.
  • Integración con IDEs: Cómo utilizar Git desde el propio IDE, como en IntelliJ IDEA, Eclipse o Visual Studio Code.

Depuración de aplicaciones

  • Concepto de depuración: Qué es depurar, tipos de errores (sintácticos, de ejecución y lógicos) y cómo detectarlos.
  • Uso de depuradores en el IDE:
    • Uso de puntos de interrupción (breakpoints).
    • Inspección de variables y memoria en tiempo de ejecución.
    • Ejecución paso a paso y monitoreo de cambios en el estado del programa.
  • Manejo de excepciones: Cómo detectar y manejar excepciones para evitar que la aplicación se detenga de forma inesperada.

Automatización de tareas

  • Herramientas de automatización:
    • Build Tools: Introducción a herramientas de construcción como Maven y Gradle (especialmente en entornos Java).
    • Scripts de automatización: Uso de scripts para tareas repetitivas, como limpieza de archivos temporales, compilación, ejecución de pruebas, etc.
  • Tareas automatizadas en IDEs: Cómo configurar y ejecutar tareas automáticas dentro del propio IDE (construcción automática, pruebas, despliegues).

Documentación de código y herramientas de generación de documentación

  • Buenas prácticas de documentación: Importancia de la documentación en el desarrollo colaborativo.
  • Comentarios en el código: Uso de comentarios para explicar funcionalidades y detalles importantes.
  • Documentación con herramientas específicas:
    • JavaDoc: Cómo documentar el código Java y generar documentación automática en formato HTML.
    • Doxygen: Herramienta para documentación en varios lenguajes de programación.
  • Control de versiones para documentación: Cómo utilizar el control de versiones también para el seguimiento y gestión de la documentación.

Pruebas y calidad de software

  • Tipos de pruebas: Pruebas unitarias, de integración, de aceptación, etc.
  • Pruebas unitarias:
    • Introducción a las pruebas unitarias y su importancia en el desarrollo.
    • Uso de frameworks de prueba como JUnit (para Java) o Pytest (para Python).
  • TDD (Test-Driven Development): Desarrollo basado en pruebas, cómo escribir pruebas antes de desarrollar el código.
  • Cobertura de código: Qué es la cobertura de código y cómo medir cuántas líneas de código son cubiertas por las pruebas.

Metodologías y desarrollo de software

  • Metodologías ágiles:
    • Introducción a metodologías como Scrum y Kanban.
    • Conceptos básicos: sprints, roles en Scrum (product owner, scrum master, equipo de desarrollo), reuniones diarias (stand-ups).
  • Planificación y gestión de proyectos:
    • Herramientas de gestión de proyectos: Trello, Jira.
    • Uso de tableros Kanban para organización de tareas.

Integración y despliegue continuo (CI/CD)

  • Integración Continua (CI):
    • Concepto de integración continua, y su importancia en equipos de desarrollo.
    • Herramientas de CI: Jenkins, Travis CI, GitHub Actions.
  • Despliegue Continuo (CD):
    • Despliegue automático de aplicaciones, cómo se configuran pipelines de despliegue.
    • Pruebas y validación antes del despliegue en producción.

Herramientas y recursos

  • IDEs: Eclipse, IntelliJ IDEA, Visual Studio Code, NetBeans.
  • Sistemas de control de versiones: Git, GitHub, GitLab.
  • Frameworks de pruebas: JUnit para Java, Pytest para Python.
  • Gestores de proyectos: Trello, Jira.
  • Herramientas de documentación: JavaDoc, Doxygen.
  • Automatización y CI/CD: Jenkins, Travis CI, GitHub Actions.

Objetivos

  • Configurar y personalizar su entorno de desarrollo.
  • Utilizar sistemas de control de versiones de forma colaborativa.
  • Depurar y realizar pruebas unitarias a sus aplicaciones.
  • Automatizar tareas y trabajar en entornos de CI/CD.
  • Documentar adecuadamente el código y gestionar proyectos con metodologías ágiles.
